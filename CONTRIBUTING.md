# Contributing to PipeShare

First off, thank you for considering contributing to PipeShare! It's people like you that make PipeShare such a great tool.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issue Tracker](https://github.com/SavasTanriverdi/PipeShare/issues) to see if someone else has already raised it. If not, feel free to open a new issue using one of our templates.

## Code formatting

PipeShare is written in Rust. We follow the standard rust formatting rules.

Before submitting a pull request, please make sure your code passes formatting and linting:

```bash
cargo fmt --all
cargo clippy -- -D warnings
```

## Pull Request Process

1. Fork the repo and create your branch from `master`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Update the documentation where relevant.
5. Create a Pull Request summarizing your changes.

Thanks for your help!
